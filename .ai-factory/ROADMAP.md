# Project Roadmap

> Orchestrator — AI-driven multi-agent pipeline that autonomously plans, implements, and reviews code milestones.

## Hardening

- [x] **Caffeinate no-crash on non-macOS** — `_with_caffeinate` in `main.py` always spawns `caffeinate` via `Popen`, raising `FileNotFoundError` immediately on Linux. Wrap the `Popen` call with `try/except FileNotFoundError`: on failure, run the wrapped function directly without caffeinate and return the elapsed string. No behavior change on macOS. [3m 4s]

- [x] **Fix stderr pipe deadlock in `_run_claude`** — `stderr=subprocess.PIPE` combined with `proc.stderr.read()` called only after `proc.wait()` can deadlock if `claude` writes > ~64 KB to stderr before exiting: stderr pipe fills, subprocess blocks, our stdout loop stalls. Fix: replace `stderr=subprocess.PIPE` with `stderr=subprocess.STDOUT` to merge stderr into the stdout stream (already captured line by line). Remove the post-wait `proc.stderr.read()` call and the `stderr` variable from all downstream uses (`RuntimeError` messages). Touch `agents.py` only. [3m 14s]

- [x] **Planner Step 3 — kill dead Explore/Task sub-agents** — `orchestrator/prompts/planner.md` Step 3 ("Explore Codebase") tells the planner to launch 2-3 `Task(subagent_type: Explore)` agents, but the planner session's toolset is `["Read","Write","Glob","Grep","Bash"]` (`agents.py:249`) with no `Task` — the primary path is dead and every run silently takes the `Fallback: If Task tool is unavailable` branch, while Step 1 in the same file already does direct `Glob/Grep/Read`. Rewrite Step 3 to drill deeper with `Glob/Grep/Read` (architecture/modules, existing patterns, dependencies), delete the three `Task(...)` templates and the `Fallback:` line, and keep the synthesis bullets (files to create/modify, patterns, dependencies, risks). Touch `planner.md` only — do NOT modify `agents.py` or the sibling prompts (`test-planner.md`/`reviewer.md`/`implementer.md`), and leave the externally-passed plan path, `## Settings` hardcode, commit-prefix ban, and `done` rule intact. Spec: `.ai-factory/notes/02-planner-step3-direct-exploration.md`. [3m 21s]

- [x] **Prompts: follow mentions from the contract line** — Both agent prompts carry fixed checklists (`planner.md` Step 0: `DESCRIPTION`/`ARCHITECTURE`/`RULES`; `reviewer.md` Context Gates: `ARCHITECTURE`/`RULES`/`ROADMAP`) and nothing instructs going from the milestone's own line out along its links — the `Spec:` note is read only opportunistically; live incident: three review rounds issued semantic blockers without ever opening the phase's governing spec. Prompt-only fix, one rule instead of a document list: **lift the context tree from your contract line — follow mentions**. The `Spec:` tag is a mention; what the note mentions are next-level mentions; `Governing spec:` in the phase header is a mention seen when reading your line in the roadmap. Depth self-limits: only links reachable from your task — never a notes-directory sweep. `planner.md` + `test-planner.md` Step 0 gain the follow-mentions block; `reviewer.md` gains one matching gate (findings judged against the lifted tree, not the roadmap line alone; plan review holds only the plan path — recover the root by matching the plan's `# Plan:` title in `ROADMAP.md`/`ROADMAP_TESTS.md`, skip the gate if no line matches). No authority/ordering language; graceful no-op on single-tier roadmaps; no Python changes; `implementer.md` untouched. Spec: `.ai-factory/notes/03-follow-mentions.md`. [4m 30s]

- [x] **Prompts: drop DESCRIPTION.md reads** — DESCRIPTION.md is retired across the toolchain: it duplicated the target project's CLAUDE.md, which the claude CLI injects into every agent run unconditionally — the facts arrive for free. Delete the DESCRIPTION.md block from `planner.md` + `test-planner.md` Step 0 (`ARCHITECTURE.md`/`RULES.md` reads stay; do NOT add a "read CLAUDE.md" instruction — it's already in context); reword `planner.md` Step 1's skip condition ("Skip if DESCRIPTION.md provides sufficient context" → context already in hand); update `docs/target-project.md` (drop the DESCRIPTION requirement; ROADMAP + git required, ARCHITECTURE/RULES optional) and grep-sweep all docs. Prompt+docs only — `agents.py`/`main.py` expected untouched, verify by grep. Target-project DESCRIPTION.md files are not deleted — reads retire, files die naturally. Spec: `.ai-factory/notes/04-drop-description-reads.md`. [6m 22s]

- [x] **Prompts/docs: drop the sweep-guard clause; specs/ naming in docs** — The follow-mentions block carries a negative depth guard ("do not sweep the notes directory or read specs of unrelated tasks") — redundant over-protection: the positive rule (lift the tree from your contract line by mentions) self-limits depth **by construction**, and the name-bound prohibition already drifted (`notes/` → `specs/` lazy migration). Delete the guard clause from `planner.md`/`test-planner.md` and from `reviewer.md`'s gate, **replacing it with nothing** — the positive rule stays word-for-word. Separately, grep-sweep `docs/` (`context-model.md` "паутина нот", `target-project.md`) to state the pair where the directory is named as the specs' home: `specs/` current, `notes/` legacy served via the tags' literal paths. No Python (verify by grep); no migration instructions for target projects. Spec: `.ai-factory/notes/05-specs-wording.md`. [5m 29s]

- [x] **Auto-push after milestone commit** — `_git_commit` in `main.py` only committed; add `git push -u origin HEAD` right after the commit, warning and continuing (not raising) on push failure, so completed milestones reach the remote without a manual step.

- [x] **Re-review branch in `PlannerReviewer.review()`** — `review()` (`agents.py:283`) sends the same generic prompt on every pass while its session is persistent; live incident: the reviewer re-asserted an already-fixed finding as "unaddressed" for two passes and killed the run at max_iterations on a false failure. Mirror `plan()`'s pattern: add `prev_review_path`; when the session already exists, the prompt names the previous review and forces per-finding re-verification (re-read the cited file, quote current lines, verdict Fixed/Not fixed) before the normal full review. `main.py` passes iteration-1's review file. Session stays persistent — prompt-level fix; `PlanReviewer` untouched. Document the failure mode in `docs/non-convergence.md`. Spec: `.ai-factory/specs/01-reviewer-rereview-branch.md`. [4m 50s]

- [x] **Implementer feedback path — point at the real review file** — `Implementer.implement()` continuing-prompt (`agents.py:371`) says feedback is in `{patches_dir}`, but in implement mode `patches/` is never populated (reviews go to `reviews/`; only test mode bridges into `patches/`, `main.py:637`) — the implementer finds feedback only by its own initiative. Replace with an explicit `feedback_path` param: implement mode passes the failed review file (`reviews/{seq}-{slug}-review-{n}.md`), test mode passes the original test-run file from `test-runs/` directly — and retire `patches/` entirely: delete the copy-bridge (`main.py:637`), the dir creation in both modes, and the first-call `patches_note` scan. Resume behavior for both modes must not regress; docs sweep (`CLAUDE.md`, `docs/test-mode.md`, `docs/how-it-works.md`); target projects' leftover `patches/` dirs are not deleted. Spec: `.ai-factory/specs/02-implementer-feedback-path.md`. [12m 3s]

- [x] **`## Deferred observations` section in `reviewer.md`** — reviewers produce verified-but-deprioritized observations ("latent", "forward risk", "flagging for Phase N") as ad-hoc Notes that no one reads on success and `roadmap-prune` later deletes; no delivery channel to the downstream phases they address. Add a standardized `## Deferred observations` section to `prompts/reviewer.md` Output Format (entry: `- Affects: <phase/spec-note/"unknown"> — <observation>`), worded role-neutrally — one prompt serves both `PlanReviewer` and `PlannerReviewer`, so `plan-reviews/` and `reviews/` gain it from one edit. Critical: extend the PASS rules so deferred observations are not findings — a review with only them still passes; section sits before the signal line; no Python changes. Anti-laundering guard: deferral criterion is **scope, not severity** — anything introduced by the current diff or fixable within the milestone's boundary is a finding down to cosmetics; only fixes lying outside the milestone (other phase, not-yet-existing consumer) may be deferred. Status field after each entry is reserved for downstream tooling — the reviewer never writes or imitates processing marks. Harvest consumer lives in the skills repo (`roadmap-prune`), not here. Spec: `.ai-factory/specs/03-deferred-observations-section.md`. [9m 11s]

- [x] **Docs sweep + config-path literals** — Docs audit against current code: `docs/target-project.md` gains the auto-push behavior (`git push -u origin HEAD` after each milestone commit, warn-and-continue on failure); `docs/how-it-works.md` lists the full sidecar `step` set (adds `plan_review_failed:N` / `review_failed:N`); `CLAUDE.md` drops the wrong "(no review pass)" comment on the implement command.
