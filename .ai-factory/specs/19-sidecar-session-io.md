# Sidecar Session I/O — Test Plan

**Date:** 2026-07-13
**Source:** roadmap-test-coverage agent

## Source Overview

`_read_sessions`, `_write_session`, and `kill_active_child` in `orchestrator/agents.py` (lines 19-66) manage the JSON "sidecar" file that tracks per-milestone agent session IDs and pipeline step state (e.g. `{"planner": "<session-id>", "implementer": "<session-id>", "step": "..."}`), plus the SIGKILL-based teardown of the in-flight Claude CLI child process group. The sidecar path is always derived from a plan file's path via `plan_path.with_suffix('.json')` (e.g. `01-foo-plan.md` → `01-foo-plan.json`). These functions are the persistence backbone for resuming an interrupted pipeline run (`orchestrator/resume.py`, `orchestrator/main.py`) and for safely interrupting an active Claude CLI subprocess on Ctrl-C.

## Instantiation

- `_read_sessions` / `_write_session`: pure functions of `Path` + `json` + `os.replace`. Test directly with pytest's `tmp_path` fixture — no mocking needed. Build a `plan_path = tmp_path / "01-foo-plan.md"` (the file itself need not exist; only its `.json` sibling matters) and inspect/pre-seed `tmp_path / "01-foo-plan.json"`.
- `kill_active_child`: reads/writes the module-level global `state.active_proc` (`orchestrator/state.py`, a bare `subprocess.Popen | None` field, no encapsulation). Tests must set `state.active_proc` directly before calling, and should reset it to `None` in a `finally`/fixture teardown to avoid leaking state across tests (it's a global, not per-instance).
  - No-op branches (`None`, or already-exited process) need no real subprocess — a `unittest.mock.Mock(poll=...)` or a small dummy object with a `.poll()` method suffices, OR a real short-lived process that's `wait()`-ed first.
  - The live-process branch needs a **real** subprocess to observe actual death: spawn with `subprocess.Popen(["sleep", "5"], preexec_fn=os.setsid)` (or `start_new_session=True`, matching production usage in `_run_claude`) so it has its own process group for `os.killpg` to target. After calling `kill_active_child()`, poll (with a short retry loop, since SIGKILL delivery isn't synchronous) until `proc.poll() is not None`, then assert `state.active_proc is None`.

## Existing Coverage

None. `tests/test_agents.py` only covers `_has_signal` and `TestRunner._extract_test_command`. `tests/test_main.py` references `_read_sessions` twice (lines 360, 734) but only in comments explaining *why* no sidecar JSON is set up for unrelated `_detect_milestone_step`/`_detect_test_milestone_step` tests — it never calls `_read_sessions`, `_write_session`, or `kill_active_child` directly. `grep -rn "_read_sessions\|_write_session\|kill_active_child" tests/` confirms no direct unit test exists for any of the three functions.

## Test Cases

### `_read_sessions`

- **should return `{}` when the sidecar file does not exist** — exercises `_read_sessions`. Setup: `plan_path = tmp_path / "01-foo-plan.md"`; do not create `01-foo-plan.json`.
- **should return the parsed dict when the sidecar file contains valid JSON** — exercises `_read_sessions`. Setup: write `{"planner": "sid-123", "step": "planned"}` to `tmp_path / "01-foo-plan.json"`; assert the returned dict equals it exactly (keys/values round-trip).
- **should return `{}` silently when the sidecar file contains malformed JSON** — exercises `_read_sessions`. This is the risky silent-failure path: a corrupted sidecar (e.g. truncated write, `"{not valid json"`) is swallowed via `except (json.JSONDecodeError, OSError)` and treated identically to "file doesn't exist," with no warning surfaced. Setup: write raw garbage text to the `.json` sidecar; assert result is `{}` (not an exception).
- **should derive the sidecar path via `.with_suffix('.json')` regardless of the plan file's original suffix** — exercises `_read_sessions`. Setup: use a `plan_path` like `tmp_path / "03-bar-plan.md"`, write the sidecar to `tmp_path / "03-bar-plan.json"` (not `.md.json`), and confirm `_read_sessions` finds it — proves the suffix-replacement (not append) contract that callers in `main.py`/`resume.py` rely on.

### `_write_session`

- **should create a fresh sidecar containing only the new key when no sidecar exists yet** — exercises `_write_session`. Setup: fresh `tmp_path`, no pre-existing `.json`. Call `_write_session(plan_path, "planner", "sid-abc")`; assert the sidecar now exists and `json.loads(sidecar.read_text()) == {"planner": "sid-abc"}`.
- **should merge the new key into an existing sidecar without clobbering other keys** — exercises `_write_session`. Setup: pre-seed sidecar with `{"planner": "sid-abc", "step": "planned"}`; call `_write_session(plan_path, "implementer", "sid-xyz")`; assert result is `{"planner": "sid-abc", "step": "planned", "implementer": "sid-xyz"}` — mirrors the real call sequence in `PlannerReviewer.plan`/`.review` (writes `"planner"`) followed by `Implementer.implement` (writes `"implementer"`) against the same sidecar.
- **should overwrite the value of an existing key when writing the same key again** — exercises `_write_session`. Setup: pre-seed `{"planner": "sid-old"}`; call `_write_session(plan_path, "planner", "sid-new")`; assert `{"planner": "sid-new"}` — matches `PlannerReviewer.review` re-writing `"planner"` after a re-review continues the same session.
- **should silently fall back to a fresh dict (dropping prior data) when the existing sidecar is corrupted JSON** — exercises `_write_session`. Confirmed from source: the `try/except (json.JSONDecodeError, OSError): data = {}` fallback means any prior keys in a corrupted sidecar are permanently lost — the write proceeds with only the new key, no error raised. Setup: write garbage text to the sidecar first, then call `_write_session(plan_path, "step", "planned")`; assert the sidecar now contains **only** `{"step": "planned"}` (not the garbage, not a merge/recovery of any kind). Worth flagging in review — this is a real data-loss silent-failure surface.
- **should write via a temp file and atomically replace, leaving no `.tmp` file behind after a successful write** — exercises `_write_session`. Setup: fresh `tmp_path`; call `_write_session`; assert `plan_path.with_suffix('.json.tmp')` does **not** exist afterward (only the final `.json` does) — verifies the `tmp.write_text(...); os.replace(tmp, p)` pattern completed and didn't leak the intermediate file.
- **should produce sidecar content that is valid, re-readable JSON via `_read_sessions`** — exercises `_write_session` + `_read_sessions` together as a round-trip check. Setup: call `_write_session` twice with different keys, then call `_read_sessions` on the same `plan_path` and assert both keys are present — cheap integration check that the two functions agree on path derivation and format.

### `kill_active_child`

- **should be a no-op and leave `state.active_proc` as `None` when `state.active_proc` is already `None`** — exercises `kill_active_child`. Setup: explicitly set `state.active_proc = None` before calling (don't rely on prior test state, since it's a shared global); call `kill_active_child()`; assert no exception and `state.active_proc is None`.
- **should clear `state.active_proc` without attempting to signal when the tracked process has already exited** — exercises `kill_active_child`. Setup: spawn a trivial real process (e.g. `subprocess.Popen(["true"])` or `["sleep", "0"]`), call `.wait()` on it so `poll()` returns a non-`None` exit code, assign it to `state.active_proc`, then call `kill_active_child()`; assert `state.active_proc is None` and no exception (covers the `proc.poll() is not None` short-circuit branch — no `os.killpg` call needed/reachable here).
- **should kill the live process group via `os.killpg(..., SIGKILL)` and clear `state.active_proc` when the process is still running** — exercises `kill_active_child`. Setup: `proc = subprocess.Popen(["sleep", "5"], preexec_fn=os.setsid)` (own process group, matching `_run_claude`'s `start_new_session=True` usage), set `state.active_proc = proc`, call `kill_active_child()`, then poll in a short loop (e.g. up to ~1s) until `proc.poll() is not None`; assert the process died (non-`None`, killed-by-signal returncode) and `state.active_proc is None`. Clean up with `proc.wait()` to avoid a zombie/reap warning.
- **should fall back to `proc.kill()` and still clear `state.active_proc` when `os.killpg` raises `ProcessLookupError`/`PermissionError`** — exercises `kill_active_child`'s except branch. Non-obvious setup: hard to trigger for real (would need a process whose pgid vanished between the `poll()` check and the `killpg` call, or a permission boundary) — recommend a lightweight fake/mock here instead of forcing a real race: an object with `.poll()` returning `None` and `.pid` set, monkeypatching `os.killpg` to raise `ProcessLookupError`, and a mock `.kill()` method; assert `.kill()` was called and `state.active_proc is None` afterward. This is the one sub-branch where the "real subprocess" approach used elsewhere isn't practical.

## Refactor Required

Testability review verdict: `kill_active_child` reads and mutates the module-level global `state.active_proc` directly instead of taking the process handle as a parameter, forcing tests to monkeypatch `orchestrator.state` rather than inject a fake `Popen`.

Note: this global-flag pattern is the project's deliberately chosen architecture (`.ai-factory/ARCHITECTURE.md` names `state.py` explicitly as "Global flag... may be imported from any layer", and the already-existing `_run_summary` tests in `tests/test_runtime.py` already test through this exact global via save/restore try/finally). A DI-style refactor (e.g. `kill_active_child(proc: subprocess.Popen | None = None)` defaulting to `state.active_proc` when not supplied) is possible but would be a deviation from the established pattern, not a fix to a defect. The test cases in this note already show the global can be tested directly via monkeypatch/reset — flagging this refactor for the decomposition stage to weigh, not asserting it's required.

## Gotchas

- **`state.active_proc` is a bare module-level global**, not encapsulated behind an accessor — tests that set it must reset it afterward (fixture `finally`/`yield` pattern), or leftover state will silently corrupt unrelated tests that also touch `kill_active_child` or `_run_claude`.
- **`_write_session`'s corruption fallback silently discards prior sidecar data.** This is arguably a real bug/risk (a genuinely corrupted sidecar mid-run would lose the `"planner"` session ID on the very next `_write_session("implementer", ...)` call), not just a quirk to characterize — worth calling out explicitly in the test names/docstrings so a future reader understands this is *observed*, not necessarily *desired*, behavior.
- **`_read_sessions` and `_write_session` both swallow `OSError` too, not just `JSONDecodeError`** — e.g. a permissions error reading the sidecar also silently degrades to `{}`/fresh-dict rather than surfacing. Only the JSON-decode-error path is easily testable via `tmp_path`; the `OSError` path would need something like chmod-ing the file unreadable (platform-dependent, likely skip on CI/Windows) — call this out as a known gap rather than forcing brittle coverage.
- **The `.tmp` suffix collision:** `p.with_suffix('.json.tmp')` on a path already ending in `.json` produces `<name>.json.tmp` correctly (Path.with_suffix replaces only the last suffix), but if a caller ever passes a `plan_path` that already ends in `.json` this could double up unexpectedly — not currently a real call site issue (all call sites pass `*-plan.md` paths) but worth a path-derivation test to pin the contract.
- **`kill_active_child` is also called from `_run_claude`'s `KeyboardInterrupt` handler** (line 167) as the only other call site besides direct test invocation — the SIGINT-driven real-world path is out of scope for these unit tests (that's `_run_claude`'s subprocess orchestration, which is loud/non-deterministic per `test-philosophy` and not unit-testable here), but it explains why `kill_active_child` must be safe to call with no active process (the no-op branches are load-bearing for that handler, not just defensive dead code).
- **No test currently exercises `_read_sessions`/`_write_session`/`kill_active_child` at all** — the two `test_main.py` comment references (lines 360, 734) are explanatory prose about *why no sidecar is set up* for unrelated detector tests, not actual coverage; do not mistake them for existing tests when scoping this work.
