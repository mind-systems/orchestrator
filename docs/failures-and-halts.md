# Отказы и остановки

Прогон оркестратора заканчивается одним из исходов: успех, **отказ** или **остановка**. Отказ и остановка — разной природы, и это различие — смысловое ядро всего поведения при сбоях.

## Отказ ≠ остановка

**Отказ** — task не сошёлся. Каждый цикл ревью ограничен конечным бюджетом попыток; если PASS-подпись не появилась в бюджете, task объявляется несошедшимся. Это **суждение о работе**: оркестратор дал ей отведённые попытки, и она не сошлась.

**Остановка** — прогон прекращён по причине, не связанной с корректностью работы: исчерпан внешний ресурс, инфраструктурный сбой не удалось поглотить, или оператор остановил прогон руками. Это **не суждение о работе** — текущий task мог быть в полном порядке.

Одно и то же прекращение прогона попадает в одну из двух категорий по единственному признаку: вынес ли оркестратор вердикт о работе, или прервался, не дойдя до вердикта.

## Инварианты

1. **Бюджет попыток конечен.** У цикла ревью — плана и кода — фиксированный лимит итераций. Его исчерпание без PASS-подписи и есть отказ; иного способа «провалить» task нет.
2. **Отказ и остановка сигнализируются по-разному.** Исход виден по цвету уведомления, не по факту «прогон закончился». Привязку цвета к типу события и как включить уведомления — см. [Конфигурация](configuration.md).
3. **Транзиентное поглощается.** Восстановимые сбои — перегрузка API, кратковременная сетевая ошибка — не эскалируют сразу: обращение повторяется ограниченное число раз. Наружу как остановка выходит только устойчивый сбой.
4. **Любая остановка восстановима.** Прерванный прогон — по любой причине — оставляет на диске состояние, достаточное, чтобы следующий запуск продолжил с последней завершённой фазы, а не с начала task. Механика resume описана в [Как это работает](how-it-works.md).
5. **Побочные эффекты fail-safe.** Сбой отправки уведомления или иного внешнего вызова никогда не меняет исход прогона — это best-effort поверх основного результата.
6. **При отказе ничего не теряется.** Несошедшийся task не помечается выполненным, а все артефакты его раундов остаются на диске для разбора. Как читать этот разбор — см. [Когда цикл не сходится](non-convergence.md).

## Причины остановки

Остановка не выносит вердикта о работе, поэтому её причины лежат вне цикла ревью:

- **Исчерпан бюджет использования** — превышен порог расхода сессии или недельного лимита. Прогон останавливается на границе task, до начала следующего. Пороги настраиваются — см. [Конфигурация](configuration.md).
- **Внешний ресурс недоступен** — лимит API исчерпан и не восстанавливается ретраями.
- **Устойчивый инфраструктурный сбой** — ошибка, которую не удалось поглотить повтором.
- **Ручная остановка оператором** — прогон прекращён извне. Мягкая остановка даёт текущему task завершиться; форсированная прекращает его немедленно.

Во всех случаях остановки исход — жёлтый: работа не признана ни успешной, ни отказавшей.
