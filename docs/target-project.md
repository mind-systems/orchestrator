# Требования к целевому проекту

## Что нужно подготовить

Оркестратор работает с любым проектом при условии что в нём есть несколько файлов в `.ai-factory/` и инициализированный git-репозиторий.

`ROADMAP.md` — список milestone-ов в формате markdown-чекбоксов. Оркестратор читает их сверху вниз и берёт первый непомеченный. Выполненные помечаются `[x]` автоматически после коммита.

`ROADMAP_TESTS.md` — аналогичный файл для тест-задач, читается только в режиме `test`. Отдельный файл позволяет держать тест-задачи отдельно от основного роадмапа.

`ARCHITECTURE.md` — опциональный файл с детальным описанием структуры модулей, границ слоёв и правил зависимостей. Если он есть, и Planner и Implementer используют его как жёсткое требование к размещению файлов.

`RULES.md` — опциональный файл с короткими проектными правилами. Каждое правило воспринимается агентами как обязательное требование, не рекомендация.

## Формат milestone-ов

Строка milestone должна выглядеть так:

```
- [ ] **Название** — Описание того что нужно сделать
```

Название используется как заголовок плана и сообщение git-коммита. Описание читает Planner при составлении плана — чем оно конкретнее, тем лучше результат. Размытые описания типа "улучшить производительность" дают размытые планы.

## Фазы

Milestone-ы можно группировать по фазам с помощью `##` или `###` заголовков:

```markdown
## Phase 1 — Core types

- [ ] **Task A** — ...
- [ ] **Task B** — ...

## Phase 2 — Services

- [ ] **Task C** — ...
```

Внутри одной фазы планировщик держит одну Claude-сессию через все milestone-ы — не тратит токены на повторное чтение `ARCHITECTURE.md` и контекста предыдущих планов. На границе фазы сессия сбрасывается.

Фазы — необязательны. Роадмап без заголовков работает нормально: все milestone-ы тогда идут в одной непрерывной сессии. Смысл добавлять фазы появляется когда в роадмапе 5+ связанных задач подряд, а между группами контекст существенно меняется.

## Брейкпоинт

Если нужно остановить оркестратор после определённого milestone, вставьте маркер `---STOP---` в ROADMAP.md:

```markdown
- [ ] **Task A** — ...
- [ ] **Task B** — ...

---STOP---

- [ ] **Task C** — ...
- [ ] **Task D** — ...
```

Оркестратор выполнит Task A и Task B, затем остановится. Task C и Task D не будут тронуты. При запуске в лог выводится сообщение что обработка остановлена по брейкпоинту.

Чтобы продолжить — уберите маркер из файла и запустите оркестратор снова.

## Git-репозиторий

После каждого milestone оркестратор делает `git add -A` и коммит. Репозиторий должен быть инициализирован и не иметь конфликтов перед запуском. Незакоммиченные изменения оркестратор не трогает — он добавляет поверх них.
