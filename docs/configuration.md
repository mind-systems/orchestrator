# Конфигурация

## Лимиты итераций

Все циклы в оркестраторе — план-ревью, implement-ревью, refactor — используют один лимит: `ORCHESTRATOR_MAX_ITERATIONS` (default 3). Это единственная переменная окружения для управления глубиной работы агентов.

## Модели агентов

Каждый агент использует свою модель и уровень усилий. PlannerReviewer работает на Opus с высоким усилием — он принимает архитектурные решения и глубоко анализирует код. Implementer работает на Sonnet с высоким усилием — ему нужна скорость при написании кода. PlanReviewer работает на Opus с высоким усилием — каждый раз в свежей сессии, независимый взгляд на план. RefactorPlanner работает на Opus с высоким усилием — аудит и верификация в одной сессии.

Модели передаются при создании агентов в `process_milestone()` и `process_refactor_milestone()`. Менять их можно напрямую в коде — конфигурация через env-переменные для моделей не предусмотрена.
