# Конфигурация

## Лимиты итераций

Все циклы в оркестраторе — план-ревью, implement-ревью, refactor — используют один лимит: `ORCHESTRATOR_MAX_ITERATIONS` (default 3). Это единственная переменная окружения для управления глубиной работы агентов.

## Модели агентов

Каждый агент использует свою модель и уровень усилий. PlannerReviewer работает на Opus с высоким усилием — он принимает архитектурные решения и глубоко анализирует код. Implementer работает на Sonnet с высоким усилием — ему нужна скорость при написании кода. PlanReviewer работает на Opus со средним усилием — он ревьюит план итеративно, каждый раз в свежей сессии.

Модели передаются при создании агентов в `process_milestone()` и `process_refactor_milestone()`. Менять их можно напрямую в коде — конфигурация через env-переменные для моделей не предусмотрена.

## Состояние между запусками

Оркестратор сохраняет своё состояние в `.ai-factory/orchestrator-state.json` в целевом проекте. Сейчас в нём хранится список ревью-файлов, созданных в текущем implement-проходе. Этот файл можно безопасно удалить — оркестратор пересоздаст его при следующем запуске, единственное последствие: перед review-проходом будут удалены все ревью, а не только новые.
