# Модель контекста: дерево восприятия агента

Конвейер передаёт агенту минимум: контракт-строку milestone и путь к роадмапу. Весь остальной контекст агент поднимает сам, следуя по связям из этой строки. Страница описывает, из чего складывается эффективное восприятие агента и почему оно собирается лениво — деревом вокруг задачи, а не графом всего проекта.

## Что агент получает буквально

- **Планировщик** — заголовок и описание milestone (контракт-строка целиком) плюс строку `Roadmap: <path> (line N)`.
- **План-ревьюер** — путь к файлу плана; свежая сессия на каждую попытку.
- **Код-ревьюер** — путь к плану; работает в сессии планировщика и наследует всю его разведку.
- **Имплементер** — путь к плану; держит одну сессию через весь fix-цикл.

Соседние задачи, спек-ноты и архитектурные документы в промпт не вкладываются.

## Рёбра, по которым поднимается контекст

Эффективный контекст собирается проходом по связям:

- **`CLAUDE.md` целевого проекта** — единственный безусловный канал: Claude CLI подхватывает его как project context при каждом запуске каждого агента.
- **`DESCRIPTION.md`, `ARCHITECTURE.md`, `RULES.md`** — обязательное чтение по Step 0 промптов планировщика.
- **`Spec:`-тег в контракт-строке** — путь к спек-ноте задачи; нота ссылается на смежные ноты, и агент идёт по этим ссылкам. Паутина нот — главный носитель межзадачных связей.
- **Роадмап по переданному пути** — читая свою строку, агент видит соседние milestone-ы, заголовок фазы и её `Governing spec:`; ревьюер сверяет изменения с роадмапом по инструкции.
- **Сессия внутри milestone** — код-ревьюер наследует контекст планирования, имплементер накапливает контекст правок.

## Дерево вместо графа

`enable_phase_sessions` выбирает между двумя стратегиями подъёма контекста:

- **`true` — жадная загрузка.** Сессия планировщика переносится между milestone-ами фазы; к последнему milestone в префиксе сессии лежит весь граф фазы — все планы, ревью и tool-выхлопы. Агент видит всё, но контекст выгорает даже при миллионе токенов, а стоимость растёт сверхлинейно (измерения — в [phase-sessions.md](phase-sessions.md)).
- **`false` (дефолт) — ленивый подъём.** На каждый milestone в память поднимается только дерево вокруг него: корень — контракт-строка, ветви — спек-нота со своими ссылками, роадмап-соседи, архитектурные документы. Межзадачная память живёт не в сессии, а в файлах — артефактах, нотах, роадмапе — и достаётся по запросу.

Ленивое дерево масштабируется с ростом роадмапа: глубина подъёма определяется связностью нот, а не длиной накопленной истории. Жадный граф — нет: его префикс растёт с каждым milestone независимо от того, нужна ли эта память текущей задаче.

## Следствия для целевого проекта

- **Контракт-строка — единственный гарантированный вход.** Всё существенное для задачи должно быть достижимо из неё по ссылкам: `Spec:`-тег, ссылки ноты на смежные ноты, `Governing spec:` фазы.
- **`CLAUDE.md` — канал для правил, действующих всегда.** Что должно быть в голове агента на каждом запуске без исключений — живёт там, не в нотах.
- **Широта восприятия равна плотности связей.** Нота, не ссылающаяся на смежные ноты, оставляет агента слепым к ним; полнота ленивого дерева в нужных местах определяется качеством рёбер, а не объёмом сессии.
